## What the MVP can do tonight

1. Log in via Supabase Auth UI (email/password OR magic link). Sign-up tab is hidden by configuration (D9 of design.md).
2. Land at `/transactions` showing the most recent 25 transactions of the authenticated user, ordered by `occurred_at DESC`.
3. Filter by account (dropdown) and by category (chips). Filters reflect in the URL as query params (`?accountId=...&categoryIds=...&page=N`).
4. Paginate forward/backward (no total exposed — see spec scenario).
5. Click a transaction's category chip → modal opens → pick another category → Confirm → `UPDATE myfinance.transactions SET category_id = $1`. The list refreshes via React Query invalidation.
6. Modal is **idempotent**: pick the same category and Confirm → no SQL fires, modal closes.
7. RLS of `backend/database/migrations/V002__rls_policies.sql` protects every read/write. The anon key is public by design.
